也是顺利结束了，博主也是成功转到了软工做预备牛马，想到之前踩过的很多坑和一点点小遗憾（虽然五题全都会但是因为一个 int 忘记转 double 的变量声明问题错过 AK 机会），我觉得还是比较有必要写这样一篇博客来告诫后人（bushi），希望能有所帮助。

机考小 Tips

  • 能用 C++ 最好用 C++，STL 库的 vector（不要再傻傻用静态数组开 9999999 个空间了，动态数组可以不用 const 啥的实现变量定义空间大小，它自带的一系列函数静态数组大多都用不了但是非常好用，还是很值得学一下的），map / set 也可以学一下在特定的环境会超级好用，然后学一下常用的 sort 等函数和迭代器，能让你速通大部分真题。
  • 学校 OJ 上的题目基本都是不告诉你数据组数的，所以我们在输入时一般会用到 while(cin >> n) 来进行多组输入，getline 也可以学一下但不常用。
  • 可以研究一下历年的真题找一找出题规律然后有针对性地去洛谷或者学校的 OJ 上进行练习效果会非常好，到我这一次考试每年固定的是第一题的输出题，和一道图形化输出题以及一道字符串处理问题，其他可能会有简单的判断设计题或者数组排序。
  • 提交的时候记得把语言改成 C++，默认是 C 直接提交会报错。
  • 换行习惯用 \n 的可以用，速度会快一点但其实没什么区别，但是用 endl 也不会超时，看个人习惯没必要强制用 \n。
  • 可以无脑用万能头 bits/stdc++.h，工程化 C++ 才需要各种头文件。

Problem B — 判定方程

Description

不高于 2 次的方程有通用表达为 ax² + bx + c = 0。小 A 希望有一个程序能判定方程求解情况，请你来帮帮他。（a, b, c 数据类型为 double）

Input

多组数据，每组数据输出系数 a, b, c 的值。

Output

根据系数判定方程解。输出分为：

  • Invalid equation（无解方程）
  • one real solution（一个实数解）
  • two different real solutions（两个不同实数解）
  • two different complex solutions（两个不同复数解）

一组输入对应一个判定结果，行末回车。

Sample Input

Sample Output

解析

数学和分支设计题，核心算法是先考虑方程无意义即 a, b = 0 的情况和只有 a = 0 的情况直接输出，前者是无解后者是一定有一个解，然后计算 Δ，根据 Δ > 0、Δ = 0、Δ < 0 三种情况进行分支判断后输出即可。

⚠️ 博主当时就是因为这道题存 Δ 的变量开了 int 导致检查半天也没发现问题，就这样改了七次水灵灵地成为前面二十几名里唯一没出 B 题的人。所以发现 WA 一定也要检查一下变量声明！！！有时候真的不是你的逻辑有问题。

Problem C — 绘制菱形

Description

小 A 最近在研究图形绘制，他想要使用指定的大写字母在屏幕上画一个菱形。请你来帮他实现这个目标。

Input

多组数据，一组数据一行，有两个数据构成。数据的第 1 个为当前的菱形高度 T（T >= 5，且 T 为小于 200 的奇数）；第 2 个数据为绘制菱形用的大写字母 C。

Output

对于每组数据，按照给定的 T 和 C 输出菱形。每行的最后一个 C 后直接回车，每个菱形之后尾随一个空行。

Sample Input

Sample Output

解析

常规图形题，把菱形分为上下两部分，可以把中间一行放到其中一部分然后找出字符和空格数量和循环时 i 的关系双重循环就行，需要注意的就是多组输出间需要换行。

Problem D — 拼数游戏

Description

小 A 热衷玩一个拼数游戏。在游戏中，他会得到数字（0~9）的牌各若干张。他按照从小到大、从 1 开始的顺序依次拼出整数（即 1, 2, 3, …）。拼出一个整数时，需要消耗对应数位上的数字牌（例如拼出 23 需要消耗一张数字 2 和一张数字 3 的牌）。当某个数所需的数字牌数量不足时，游戏结束。此时，他成功拼出的最大整数即为答案。（数据保证不存在拼不出整数 1 的情况）

Input

输入多组数据，每组数据包含 10 个非负整数，依次表示数字 0 到 9 的牌的数量。

Output

对应输入的一套牌，输出小 A 能拼到的最大数字。一行一个。行末无多余空格，直接回车。

Sample Input

Sample Output

解析

数组计数题，开一个大小为 10 的数组进行初始化后从 temp = 1 开始循环，每次循环检查 temp 每一位上的数对应的数组还有没有牌即对应数组是否 > 0，如果没有了那就退出循环输出 temp。

Problem E — 左右匹配

Description

匹配检查是计算机在表达式处理中必须面对的问题。现在请你来完成这个处理工作：某个字符串（长度不超过 200）中有若干字母字符，不包含空格，可能包含字母 L、R。规定任何一个 L 从内到外与在它右边且距离最近的 R 匹配。需要标记无法匹配的 L 和 R，标记方式为输出原来的字符串，并在下一行标出不能匹配的字母。不能匹配的 L 用 $ 标出，不能匹配的 R 用 * 标出。若所有 L、R 都恰好匹配，则输出 Perfect。若输入不包含 L、R，输出 None。

Input

多组输入，每组输入一个字符串（长度不超过 200，包含若干字母字符，不包含空格，可能包含字母 L、R）。

Output

对应每组输入，输出原始字符串序列且在下一行 L、R 不能匹配位置标记 $ 和 *。都匹配，则输出 Perfect。不包含括号，则输出 None。行末回车。

Sample Input

Sample Output

解析

标记题，因为数据量很小（字符串长度不超过 200）最简单的用数组模拟标记暴力搜索也可以过，不过如果你有学过 STL 的栈（stack）的话这道题就会非常简单。当时博主由于在第二题花了太多时间只留了十几分钟写 E 但是也是写到了最后的遍历输出，可惜没时间交了 😭。

核心思路：先进行特殊判断，遍历一遍看有没有 L/R，没有直接输出 + continue 即可，如果有就再遍历一遍遇到 L 让其下标入栈，遇到 R 判断栈是否为空，若为空则标记为 *（开一个 mark 数组），否则弹栈，遍历完后判断栈若为空即输出 Perfect，否则就在 mark 里标记剩下 L 的下标，最后遍历输出即可。

总结

机考通过标准是 5 过 3，往年的通过率都是在 30% 左右但今年是有 50% 左右，进面的每年都会小于拟招人数，所以机考过了基本就可以确定转掉了，面试作用主要是看你是否需要降转。所以建议如果不想降转的同学可以看看计院的培养计划看和原专业差几门课，差的多的话可以在大一下前问问导员有没有办法退一部分课和辅修一些 C++/Python 啥的，也没必要有很高的绩点，有好几个人 2.6 左右都进了，但是单门学科的硬性要求要达到比如高数/英语/编程语言课等 70+，详见每年转专业细则。

总的来说工大转专业还是相对很宽松的，计院报的人减少可能也有一定关系，这届热门的是机械和信工，前者要求绩点很高（机械工程 29/80，我有同学绩点 3.5 都没进去），后者又要看绩点又要看机考。所以励志转计院的只要早点学 C++ 就可以了，博主是高三毕业暑假开始学的 C++，坚持一年学习下来你会发现机考的题目难度确实不大，但很考验临场能力和细节处理，总之加油吧。

💡 tips：一定要多刷题！
